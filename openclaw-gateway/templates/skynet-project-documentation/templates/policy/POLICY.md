# Project Policy (Strict)

## Execution
- All execution must flow through control-plane tasks.
- No task execution without a successful claim.
- No work on files without ownership.

## Documentation
- PRD and task_plan.md must exist.
- Plan must be FINALIZED before execution.

## Change Control
- Architecture, infra, dependency, and policy changes require approval.

## Agent Behavior
- Agents must update execution ledger and progress after completing tasks.
- Agents must log findings for pitfalls and decisions.
